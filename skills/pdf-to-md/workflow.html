PDF / Image → Markdown

pdf-to-md skill 工作流程

FAST

原生文字 PDF

pymupdf 直接提取，avg >50 ch/pg

OCR

扫描/图像 PDF

PaddleOCR v2 异步 API，avg ≤50 ch/pg

IMG

图片文件

JPG/PNG/BMP/TIFF/WEBP

1
原生文字 PDF 快速提取
pymupdf 秒级 
~/.venvs/general/bin/python \
  ~/.cc-switch/skills/pdf-to-md/scripts/pdf_to_md.py \
  --input "<source>" --output "<output>"
avg > 50 ch/pg原生 PDF ✓ → 生成 <stem>.md
avg ≤ 50 ch/pg扫描 PDF ✗ → 跳到 Step 2

可选: --large-image-kb 30 --no-images --max-images 50

嵌入大图保存到 <stem>/imgs/，正文留 ![](...) 占位符

↓
1b
填写图片描述
Claude Vision 无 API Key

Read tool 读图 → Edit 替换占位符为 Markdown 描述

≤ 5 张直接逐张读取替换
> 5 张spawn subagents（每 10-20 张）
↓
2
OCR 识别（扫描 PDF + 图片）
PaddleOCR v2 异步 API

Cloud — 仅需 PADDLEOCR_TOKEN，异步 Jobs API + multipart 上传

export PADDLEOCR_TOKEN="your_token"
python3 ~/.cc-switch/skills/pdf-to-md/scripts/ocr_extract.py \
  "<source>" "<output>"
上传multipart/form-data，本地文件 ≤ 50MB
大文件≥ 45MB PDF 自动拆分，多个 job 并发提交
进度实时反馈 进度: 12/36 页
超时单 job 30 分钟

optionalPayload 默认开启： 方向校正、去弯曲、图表识别、版面检测、跨页表格合并、标题层级、Markdown 美化

Local — 需 ~/.venvs/paddleocr，仅 PDF

~/.venvs/paddleocr/bin/python \
  ~/.cc-switch/skills/pdf-to-md/scripts/ocr_extract.py \
  "<source>" "<output>" --server_url http://localhost:8111/

ocr_extract.py 用位置参数，不是 --input/--output

<output>/<stem>/ per_page/ 逐页 MD（多页时生成） merged/ 全文合并 MD + 下载图片 pymupdf_raw/ pymupdf 并行提取原文（仅 PDF）
↓
2b
OCR + pymupdf 文字融合
Agent 融合 LLM 智能校验

核心价值：pymupdf 文字零误差但无图片内容，OCR 覆盖全文但有识别错误。两者互补，agent 融合后字准最高、覆盖最全。

pymupdf 文字丰富以 pymupdf 文字为主，OCR 文字为辅交叉校验
pymupdf 文字稀疏以 OCR 为主（扫描件/图片页）

流程：读取 merged/ (OCR) + pymupdf_raw/ (pymupdf) → 逐页对比 → Edit 写入融合结果

↓
3
可选：审查与优化
按需

读取 references/optimization-prompt.md 规范，用 Edit 修改

修正OCR 错误标注 [存疑] 或 [OCR修正]
清理页眉页脚、冗余空格、乱码
格式标题层级、引用块、列表、表格

--api-key 
~/.venvs/general/bin/python \

# PDF 内容优化规范

处理从 OCR 提取的原始文字时，按以下规范清理并输出结构化 Markdown。

## 处理优先级

1. 内容完整性与准确性
2. 结构保真（保留原文档的逻辑层级）
3. 透明化修改（所有非原文修正必须标注）
4. 格式美化与关键信息高亮
5. 补充上下文解释

## 内容编辑

- **修正 OCR 错误**：根据上下文修正识别错误。对不确定的修正使用标注：`[OCR修正：原识别为"拌析"，修正为"分析"]`
- **去除冗余内容**：清除多余空格、断行、乱码、重复文字
- **去除页眉页脚**：删除重复出现的课程标题行、页码、App 水印、版权行（如"版权归得到App所有，未经许可不得转载"）
- **高不确定性内容**：无法判断时使用 `[存疑：原文为"..."，含义不明确，当前按"..."处理]`

## 内容优化

- **关键信息加粗**：关键定义、核心论点、重要数据、结论性语句使用 `**粗体**`
- **统一术语**：确保全文同一概念表达一致
- **补充注释**：仅当术语首次出现、文档未作解释、且对理解至关重要时，简短补充（1-2句）

## 格式化输出

- 使用 Markdown 标题层级还原原文章节结构（`#` `##` `###`）
- 对话、问答场景使用 `>` 引用块
- 列表、步骤使用有序或无序列表
- 表格使用 Markdown 表格格式
- 最终输出必须是完整记录，不得缩写或摘要化

## OCR 常见错误模式（中文场景）

| OCR 错误 | 正确 | 说明 |
|---------|------|------|
| 杀近自然 / 杀近 | 亲近自然 / 亲近 | 形近字误识 |
| 友现 / 上友现 | 发现 | 高频误识 |
| 局友 / 局上友 | 引导 | 高频误识 |
| 束 | 就（上下文判断） | 高频误识 |
| 乔（培乔、乔宠物）| 养（培养、养宠物）| 形近字误识 |
| 夏令曹 / 令莒 | 夏令营 | 高频误识 |
| 晴旷 / 晴昨 | 蜻蜓 | 形近字误识 |
| 伦（一伦花）| 朵（一朵花）| 形近字误识 |
| 上友 | 发（发展、发现等，视上下文）| 高频误识 |

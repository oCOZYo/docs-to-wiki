1
原生文字 PDF 快速提取  pymupdf · 秒级
pdf_to_md.py — 直接提取文字 + 嵌入大图，扫描 PDF 自动跳过
FAST pymupdf ▼
只处理原生文字 PDF（avg >50 ch/pg）；扫描 PDF 输出 scanned (avg N ch/pg) 后跳过，进入 Step 2
~/.venvs/general/bin/python \
  ~/.cc-switch/skills/pdf-to-md/scripts/pdf_to_md.py \
  --input "<source_dir_or_file>" \
  --output "<output_dir>"
参数默认值说明
--large-image-kb30嵌入大图超过此值（KB）才提取，过滤小图标/徽标
--no-images—纯文字模式，跳过所有图片，零 API 消耗
--max-images50单文档图片数上限（费用保护）
--api-key—传入则启用 Standalone 模式，脚本自行调用 Vision
--modelDOCS_TO_WIKI_MODEL /
claude-haiku-4-5-20251001		Standalone 模式 Vision 模型
输出：
· <output_dir>/<stem>.md — 文字 + ![](…) 图片占位符
· <output_dir>/<stem>/imgs/img_NNN.{ext} — 提取的嵌入图片（持久保存）
1b
填写图片描述  Agent 模式 · 仅有嵌入图时执行
Read tool 读取图片 → Edit 替换 ![](...) 占位符为完整 Markdown 描述
可选 Vision ▼
≤ 5 张图片
直接用 Read tool 逐张读取
Edit 替换占位符
> 5 张 / 多文件批量
spawn subagents（每个 10–20 张）
图片字节只进 subagent 上下文，主 agent 不膨胀
描述内容：图片内容、流程图节点与箭头、图表数值与坐标轴、对比布局、完整表格
无需独立 API Key：使用 Claude Code 内置 Vision 能力
2
OCR 识别  PaddleOCR v2 异步 API
扫描 PDF + 图片文件；云端 Jobs API + multipart 上传；大文件自动拆分
API 异步 ▼
接口注意：ocr_extract.py 使用位置参数（无 --input/--output 标志）
export PADDLEOCR_TOKEN="your_token"
python3 ~/.cc-switch/skills/pdf-to-md/scripts/ocr_extract.py \
  "<source_dir>" "<output_dir>"
参数默认说明
--modelPaddleOCR-VL-1.5OCR 模型选择
--pages—页码范围，如 "2,4-6"
--no-chart—禁用图表识别
--no-merge-tables—禁用跨页表格合并
--max-file-mb45大文件自动拆分阈值（MB）
特性说明
上传方式multipart/form-data，本地文件 ≤ 50MB
大文件处理≥ 45MB PDF 自动按页拆分为多个子文件并行提交
实时进度控制台输出 进度: 12/36 页
超时单 job 30 分钟
默认开启方向校正、去弯曲、图表识别、版面检测、跨页表格合并、标题层级
输出目录结构：
· per_page/ — 逐页独立 MD（多页时生成）
· merged/ — 全文合并 MD，适合直接导入知识库
· pymupdf_raw/ — pymupdf 并行提取原文（仅 PDF 文件生成）
2b
OCR + pymupdf 文字融合  Agent 智能校验 · 有 pymupdf_raw 时执行
两路文字互补：pymupdf 零误差 + OCR 覆盖图片区域，agent 逐页对比融合
可选 Agent ▼
核心价值：pymupdf 文字零误差但无图片内容，OCR 覆盖全文但有识别错误。两者互补，agent 融合后字准最高、覆盖最全。
页面情况融合策略
pymupdf 文字丰富（>50 字符）以 pymupdf 文字为主，OCR 文字为辅交叉校验
pymupdf 文字稀疏/为空以 OCR 为主（扫描件/图片页）
流程：读取 merged/<stem>.md（OCR）+ pymupdf_raw/<stem>.md（pymupdf）→ 逐页对比 → Edit 写入融合结果到 merged/
3
审查与调整  按需 · 用户要求时执行
修正识别错误、统一术语、优化标题层级
可选 ▼
先读取 references/optimization-prompt.md 中的处理规范，再用 Edit 工具针对性修改
操作类型说明
修正识别错误标注 [存疑] 或 [OCR修正]

~/.venvs/general/bin/python \

✅ 最终输出结构
原生文字 PDF 路径
output_dir/
  ├── 文件A.md            ← 文字 + 图片描述
  └── 文件A/
      └── imgs/
          ├── img_001.png  ← 提取的嵌入图
          └── …
扫描 PDF / OCR 路径
output_dir/
  └── 文件A/
      ├── per_page/     ← 逐页 MD
      ├── merged/      ← 合并 MD ⭐
      └── pymupdf_raw/ ← pymupdf 原文
· 原生文字 PDF：无 OCR API 消耗，秒级完成
· 扫描 PDF：需 PADDLEOCR_TOKEN，实时进度反馈，单 job 最长 30 分钟
· 图片文件（<stem>/imgs/）持久保存，agent 可在任意时间点 Read
· 大文件（≥45MB PDF）自动拆分多个子文件并行提交，结果按顺序合并
· Token 含个人凭证，始终用环境变量传入，不要写在命令行

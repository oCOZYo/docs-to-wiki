--api-key—Anthropic API Key。仅用户明确要求 Standalone 模式时传入；即使环境中有 ANTHROPIC_API_KEY 也不自动使用
--modelDOCS_TO_WIKI_MODEL /

PADDLEOCR_TOKEN — AI Studio 凭证

仅用户明确要求时传 --api-key sk-ant-…

  • 扫描 .pptx / .ppsx

输入是单文件
📄 直接处理
  • 仅处理该文件

⟳ 顺序遍历每个文件
1
PPTX → PDF  LibreOffice headless
soffice 转换，独立 profile 防并发 lock；PDF 写入临时目录
LibreOffice ▼
细节值
命令soffice --headless --convert-to pdf
profile 隔离/tmp/lo_profile_pptx2md_{worker_id}
防止多文件并行时 soffice lock 冲突
超时120 秒
成功条件PDF 存在 且 大小 > 0
PDF 存储tempfile.TemporaryDirectory（处理完自动清除）
失败路径：转换失败（超时、soffice 不存在、文件损坏）→ 写入 {stem}.md 内容为 *[PDF conversion failed]* → 跳至下一文件
注：macOS 下 LibreOffice 的 --convert-to png 仅导出第一页，因此走 PDF 中间路径再用 pymupdf 逐页渲染。
2
PDF → PNG  pymupdf (fitz)
每页渲染为 PNG，持久保存到 output_dir/stem/slides/；Resume：已有正确数量则跳过重渲
pymupdf ▼
细节值
渲染矩阵fitz.Matrix(dpi/72, dpi/72) — 默认 96dpi ≈ 1.33× 原尺寸
色彩空间RGB
命名规则slide_001.png, slide_002.png …（零补位3位）
截断pngs = pngs[:max_slides]
⭐ 存储位置output_dir/{stem}/slides/ — 持久目录，脚本退出后 PNG 仍在磁盘
Resume若 slides/ 已有预期数量 PNG，跳过重渲（打印 [resume] 提示）
失败路径：PNG 列表为空 → 写入 {stem}.md 内容为 *[No slides rendered]* → 跳至下一文件
文件大小参考（96dpi）：典型幻灯片 PNG 约 150KB–1.5MB，均值约 300KB，base64 后约 400KB。 每批 8 张 ≈ 3.2MB，远低于 32MB API 上限。高 DPI（120+）或内容密集时建议减小批次（4–5 张）。
2b
OCR 所有幻灯片（并行） PaddleOCR 云端
ThreadPoolExecutor 并行调用 · PADDLEOCR_TOKEN 存在时运行；未设置则跳过，桩文件无 OCR 块
可选 API ▼
细节值
凭证来源环境变量 PADDLEOCR_TOKEN + PADDLEOCR_API_URL（不走 CLI，token 不进 shell 历史）
并发数--ocr-concurrent，默认 5
超时60 秒 / 张
请求格式{"file": base64, "fileType": 1, "useDocOrientationClassify": true, "useDocUnwarping": false}
响应路径result.layoutParsingResults[0].markdown.text
降级单张失败 → 返回空字符串，其他张继续；无 token → 整批跳过
OCR 的作用：将提取的文字注入 Vision 提示词 <ocr_text> 块， 让 Vision 聚焦于视觉结构（流程图箭头、对比布局、图表趋势）， 文字识别由 OCR 兜底——一次 Vision 调用同时获得结构和文字的高精度描述。
有 PADDLEOCR_TOKEN：
并行 OCR → ocr_map {slide_num: text}
无 PADDLEOCR_TOKEN：
跳过 OCR → ocr_map = {} → 桩文件无 ocr 块，Vision 照常运行
3
写 per_page 桩文件
每张幻灯片写 per_page/slide_NNN.md，含 sentinel + 图片绝对路径 + OCR 文字（如有）
I/O ▼
Resume：若 per_page MD 已存在且不含 sentinel（已填写），则跳过该张——保留 subagent 的工作，支持中断续跑。
<!-- pptx-to-md:stub -->
<!-- png: /Users/zen/out/deck/slides/slide_003.png -->

<!-- ocr:
顺丰统保项目背景
顺丰于2022年启动车辆全国统保...
-->
字段说明
Sentinel<!-- pptx-to-md:stub -->（首行）— Resume 检测标志；已填写文件无此行
PNG 路径绝对路径，subagent 无需推断 base dir
OCR 块有 OCR 文字则写入 <!-- ocr: … -->；无则省略该块
已填写判断file.read_bytes()[:32].startswith(b"<!-- pptx-to-md:stub -->")
✅ AGENT 模式路径
4a
Agent / Subagents 填写 per_page MDs
直接写磁盘，不返回 JSON；崩溃后已写文件保留
按幻灯片数量选择策略：

≤ 5 张：Agent 直接逐张处理
  1. Read per_page stub → 提取 PNG 路径 + OCR 文字
  2. Read PNG
  3. Write 覆写 per_page MD（纯描述，无 sentinel）

> 5 张：spawn subagents（每批 ≤ 8 张）
  图片字节只进 subagent 上下文，主 agent 不膨胀
  subagent 直接 Write 到 per_page MD
  Resume check：首行无 sentinel → 跳过（已处理）
32MB 限制：96dpi PNG 均值约 300KB，base64 后约 400KB。 每批 8 张 ≈ 3.2MB。若 DPI 更高（120+），建议减至 4–5 张/批。
5a
--merge-only 合并（所有 subagents 完成后）
排序 slide_*.md，逐张追加 ## Slide N + 图片链接 + 描述
python pptx_to_md.py \
  --input deck.pptx \
  --output ./out/ \
  --merge-only
输出 stem.md；unfilled stub 以 *[not yet described]* 占位
可多次运行，已 filled 的幻灯片不会被清除
🔧 STANDALONE 模式路径
4b
Vision + OCR 上下文（并行）
ThreadPoolExecutor；OCR 文字注入提示词；逐张写 per_page MD
有 OCR → VISION_PROMPT_WITH_OCR
无 OCR → VISION_PROMPT_NO_OCR
Resume：跳过无 sentinel 的文件
max_tokens=1024，描述写入 per_page MD
5b
自动 merge → stem.md

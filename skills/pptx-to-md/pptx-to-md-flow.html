入口 CLI
PPTX 入口 pptx_to_md.py，PDF 入口 pdf_to_md.py；图片直入 ocr_extract.py
CLI ▾
入口位置处理目标
pptx_to_md.py~/.cc-switch/skills/pptx-to-md/scripts/PPTX/PPSX → 调 LibreOffice → 转交 pdf_to_md.py
pdf_to_md.py~/.cc-switch/skills/pdf-to-md/scripts/PDF → 文字检测后分两路：pymupdf 或自动 OCR
ocr_extract.py~/.cc-switch/skills/pdf-to-md/scripts/位置参数调用；图片走这里，PDF 由 pdf_to_md.py 自动 fork 进来
本图聚焦 PPTX 路径。原始 PDF 输入跳过 Step 1 直接进入 Step 2；图片输入直接走 ocr_extract.py。
1
PPTX/PPSX → PDF
LibreOffice headless 转换 · soffice --convert-to pdf · profile 隔离防 lock
LibreOffice ▾
项说明
命令soffice -env:UserInstallation=file:///tmp/soffice_<wid> --headless --convert-to pdf --outdir <dir> <pptx>
Profile 隔离每个 worker 用独立 UserInstallation，多文件并行不互锁
超时120 秒（单文件）
中间产物临时目录（默认）或 --pdf-dir 持久化
失败处理写 *[PDF conversion failed]* 占位 .md，继续下一个文件
LibreOffice 转出的 PDF 通常带文字层（PPTX 中的文本框保留为 PDF text objects），所以 Step 2 大概率走快速路径。若 PPTX 完全由扫描图组成，则会触发 Step 2 的 OCR 分支。
2
PDF 文字密度检测（决策点）
pdf_to_md.py 用 pymupdf 计算平均字符/页，决定走哪条分支
pymupdf ▾
项值 / 说明
判定函数is_text_pdf(doc) → 返回 (is_text, avg_chars)
阈值TEXT_THRESHOLD = 50（平均字符/页 > 50 视为文字 PDF）
状态字符串text (avg N ch/pg, K images extracted) / ocr (avg N ch/pg) / scanned ... — set PADDLEOCR_TOKEN
分支决策：文字密度高 → 左侧 pymupdf 路径；密度低 → 右侧 PaddleOCR 路径（需 PADDLEOCR_TOKEN，否则跳过）。
3A · pymupdf 文字路径（快速）
逐页结构化提取
  • page.get_text("dict") 拿到 blocks
  • 用 中位数字号 作 body，按 size/body 比例推 H1/H2/H3
  • 段落间垂直距离 > 1.5×body 时插空行
  • 嵌入大图 > --large-image-kb（默认 30KB）保存到 imgs/ 留 ![](...) 占位
  • 非空白 page → 段落用 "\n\n---\n\n" 拼接
输出：<output>/<stem>.md + <output>/<stem>/imgs/img_NNN.{jpg,png,gif}。Agent 用 Read tool 读图、Edit 替换占位符。
3B · PaddleOCR fallback（自动）
PDF 直接吃，无 PNG 中间环节
  • 校验 PADDLEOCR_TOKEN；未设置则跳过并提示
  • subprocess.run(ocr_extract.py, src_tmpdir, output_dir)
  • 1800 秒（30 分钟）总超时
  • 结束后将 <output>/<stem>/merged/<stem>.md 复制为 <output>/<stem>.md（顶层入口对齐文字路径）
3B-1 multipart 上传整个 PDF（>45MB 自动按页拆 chunk）
3B-2 提交 async job → 轮询进度 extractedPages/totalPages
3B-3 并行抽 pymupdf 原文（备份 / 后续融合）
3B-4 拉 JSONL，写 merged/、per_page/、pymupdf_raw/，下载图片
4 图片描述（仅 3A 路径有 ![](...) 占位符）
Agent 模式（默认）
≤5 张：主 agent 用 Read + Edit 替换
>5 张：spawn subagents（每个 10–20 张），图片 base64 只进 subagent 上下文，节省主上下文
Standalone 模式（--api-key）
脚本自调 Claude Vision，inline 描述写入 MD
仅用户明确要求"无人值守/cron"时使用；OCR 路径不受影响
注意：3B 路径（OCR）已直接产出完整 MD，不需要图片占位填写流程。
5 · 最终产出结构
<output_dir>/
├── <stem>.md            ← 顶层入口（两条路径一致）
└── <stem>/
    ├── imgs/            ← 3A：pymupdf 抽出的嵌入大图
    ├── merged/<stem>.md ← 3B：OCR 合并文档（<stem>.md 是它的副本）
    ├── per_page/        ← 3B：每页一份 MD（>1 页时生成）
    └── pymupdf_raw/     ← 3B：pymupdf 并行抽取的原文（融合用）
融合（可选 Step 6）：3B 路径产出后，agent 可读 merged/<stem>.md 与 pymupdf_raw/<stem>.md，逐页交叉校验（pymupdf 原文优先，OCR 补图片区域文字）。
P
关键参数与环境变量
所有 PPTX-level 参数透传给 pdf_to_md.py
▾
参数 / 变量默认作用
--input—PPTX/PPSX 文件或目录
--output—输出目录
--large-image-kb30嵌入图大于此 KB 才提取（仅 3A 路径）
--max-images50单文档图片上限（费用保护）
--no-images—跳过所有图片，纯文字输出
--no-ocr-fallback—NEW 禁用 3B 路径；扫描件按"跳过"处理
--api-key—Standalone 模式：脚本自调 Claude Vision 描述图片
--modelclaude-haiku-4-5-20251001Standalone 模式的 Vision 模型
--pdf-dir—持久化中间 PDF（默认走临时目录）
PADDLEOCR_TOKEN—环境变量；设置后 3B 路径才会启用（不走 CLI，避免 token 进 shell 历史）
DOCS_TO_WIKI_MODEL—环境变量；统一覆盖 standalone 模式的 model

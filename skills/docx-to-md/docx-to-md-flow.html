💡 核心原则
DOCX 是结构化 XML，文字可直接无损提取，无需 OCR。 嵌入图片（架构图、流程图、截图）若占比较大，图文关系本身是信息——本 Skill 将图片提取到磁盘， 由 Claude agent 用内置 Vision 描述完整内容，无需单独 Anthropic API Key。
DOCX → python-docx 提取文字/表格 → 图片保存磁盘 → .md 含 ![](...) 占位符 → agent 填入描述

输出：
· <output_dir>/<stem>.md — 文字/表格 + ![](…) 图片占位符
· <output_dir>/<stem>/imgs/img_NNN.{ext} — 提取的嵌入图片（持久保存）
限制：不处理 EMF/WMF 矢量格式（python-docx 不直接暴露）；主要识别 PNG/JPEG/GIF/BMP/WEBP 嵌入图

填写图片描述  Agent 模式 · 仅有嵌入图时执行
Read tool 读取图片 → Edit 替换 ![](...) 占位符为完整 Markdown 描述
Vision 有图时执行 ▼
≤ 5 张图片
直接用 Read tool 逐张读取
Edit 替换占位符
> 5 张 / 多文件批量
spawn subagents（每个 10–20 张）
图片字节只进 subagent 上下文，主 agent 不膨胀
Subagent Prompt 模板：
对以下图片路径列表，逐一用 Read tool 读取图片，生成 Markdown 描述： 说明图片内容、流程图节点与箭头、图表数值与坐标轴、对比布局、完整表格。 返回 [{path, description}] JSON 数组。
描述内容：图片内容、流程图节点与箭头、图表数值与坐标轴、对比布局、完整表格
无需独立 API Key：使用 Claude Code 内置 Vision 能力

🔧 Standalone 模式（后台/cron 无人值守）
~/.venvs/paddleocr/bin/python \
  ~/.cc-switch/skills/docx-to-md/scripts/docx_to_md.py \
  --input file.docx --output out/ \
  --api-key sk-ant-... --model claude-haiku-4-5-20251001
也可用环境变量统一覆盖模型：export DOCS_TO_WIKI_MODEL=claude-sonnet-4-6
✅ 最终输出结构
output_dir/
  ├── 文件A.md               ← 最终 Markdown（文字/表格 + 图片描述）
  └── 文件A/
      └── imgs/

          └── …
· 文字提取无损，无 OCR 误差
· 图片文件持久保存在 <stem>/imgs/，agent 可在任意时间点 Read
· --large-image-kb 30 默认过滤掉徽标/图标等小图（30KB 以下）
· --no-images 纯文字模式时完全零 API 消耗
· 环境：~/.venvs/paddleocr/bin/python（含 python-docx）
